Springcloud第三话 —— 配置中心

Posted by 会飞的星阳 on 2019-08-01

在日常的项目开发中，我们无法避免一种情况，那就是配置文件有改动，比如说，数据库账号信息大规模变动，导致Jdbc配置需要重新修改账号信息，一般情况下，我们需要进行一系列繁琐的操作，而且还会重启处于生产环境的项目。为了优化这种行为，我们可以选择使用SpringCloud—config，因为它功能全面强大，可以无缝的和spring体系相结合。

Spring Cloud Config

之所以叫作配置中心，我们需要了解它的核心功能有什么。

  • 提供服务端和客户端支持
    • 集中管理各环境的配置文件
    • 配置文件修改之后，可以快速的生效（重点）
    • 可以进行版本管理
    • 支持大的并发查询
    • 支持各种语言

Spring Cloud Config项目是一个解决分布式系统的配置管理方案。它包含了Client和Server两个部分，server提供配置文件的存储、以接口的形式将配置文件的内容提供出去，client通过接口获取数据、并依据此数据初始化自己的应用。Spring cloud使用git或svn存放配置文件，默认情况下使用git。

接下来配置client端

client端

1.添加pom.xml依赖

2.创建两个配置文件，application.properties和bootstrap.properties
application.properties内容如下：

spring.application.name=spring-cloud-config-client
server.port=10087
#springboot 1.5.X 以上默认开通了安全认证，所以需要在配置文件添加以下配置
management.security.enabled=false

bootstrap.properties内容如下：

spring.cloud.config.name=neo-config    #对应{application}部分
spring.cloud.config.profile=dev   #对应{profile}部分
spring.cloud.config.uri=http://localhost:10086/    #对应的
spring.cloud.config.label=master  #对应git的分支

3.创建启动类

启动类只需要@SpringBootApplication注解就可以

@SpringBootApplication
public class ConfigClientApplication {

    public static void main(String[] args) {
        SpringApplication.run(ConfigClientApplication.class, args);
    }
}

4.使用@RestController注册控制器进行web测试
使用@Value注解来获取server端参数的值

@RestController
@RefreshScope // 使用该注解的类，会在接到SpringCloud配置中心配置刷新的时候
//，自动将新的配置更新到该类对应的字段中。
class HelloController {
    @Value("${neo.hello}")
    private String hello;

    @RequestMapping("/hello")
    public String from() {
        return this.hello;
    }
}

启动项目访问 ：http://localhost:10087/hello 返回配置内容即为成功

如果接下来你修改github上的配置文件内容再次进行测试，会发现读取到的配置参数还是之前的旧参数，为什么没有像我们上方标重点说的一样呢？（配置文件修改之后，可以快速的生效）

原因是我们需要手动刷新客户端配置，在win上面打开cmd执行curl -X POST http://localhost:10087/refresh， 返回 [“neo.hello”]说明已经更新了neo.hello的值。我们再次访问 http://localhost:10087/hello ，返回：hello im dev update,客户端已经得到了最新的值。
